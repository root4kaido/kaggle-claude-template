# EDA実行スキル

## 作業ディレクトリの作成

EDAは `eda/` 内に連番ディレクトリを作成し、その中で作業する。

- 命名: `eda/eda001_xxx/`, `eda/eda002_xxx/`, ...（連番 + 内容を示す短い名前）
- 既存の最大番号の次の番号を使うこと

### ディレクトリ内の構成

```
eda/edaXXX_xxx/
├── *.py              # 分析スクリプト
├── report.md         # このEDAのレポート
└── figures/          # 可視化画像
```
ユーザーに見せたいなどの意図があれば、.ipynbファイルでも構いません。

## 手順

1. **データの特定**: `$ARGUMENTS` でパスが指定されていればそれを使う。なければ `datasets/` 内のデータを探す

2. **`eda/` 内の既存ディレクトリを確認し、次の連番でディレクトリを作成**
分析観点が変わるなどした際に、気軽に次の連番に移動していただいて構いません。

3. **EDAの観点**
   EDAは実験内容と深い相互関係にあります。EDAした内容をもとに仮説を立て、精度向上のための実験を行う。その結果を用いて、またEDAを行う。このサイクルを回すことが大切です。

4. **結果を同ディレクトリ内の `report.md` に記録**。以下のフォーマットで:
   - 分析の目的
   - 分析概要
   - 発見事項
なるべく詳細に記載してください。

5. **重要な知見は `eda/eda_summary.md` にも追記する**:
   - ディレクトリ名
   - 分析の目的
   - 重要な知見の要約
   - 精度向上の仮説(もしあれば)

6. 可視化結果は同ディレクトリの `figures/` に画像として保存する
